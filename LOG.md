## [647] Erdős–Selfridge — VERIFIED (exhaustive negative)

**Key step.** `max_{m<n}(m + τ(m)) ≤ n+2` rewrites as `τ(n−j) ≤ j+2 for all j ≥ 1`.
Since `τ(m) ≤ 6720` for `m < 10¹²` (A066150), only `j ≤ W = 16384` can matter.
That makes the predicate *local*, so the range splits across cores with a
`W+2` overlap. The C program measures the largest τ it actually meets and
`progress647.py` refuses to report a bound if `max τ + 2 ≥ W` — the cutoff is
checked, not assumed.

**Cross-check.** `verify647.py` is an independent brute force (sieve τ by
counting multiples, track the running maximum, no locality argument). On
`[2, 10⁶]` it and the C search agree exactly: `{2,3,4,5,6,8,10,12,24}`.

**Throughput.** 1.75 s per 10⁸ on an idle machine at 10¹³; ~7 s per 10⁸ once ten
CPU-bound processes are competing. Observed ≈1.1×10¹¹/hour with 8 workers.
I originally aimed at 10¹³; at the real rate that is ~90 hours, so the honest
target is ~10¹². (Early on I misread `ls` output sorted alphabetically as
chronological and briefly thought the job had stalled — it had not.)

**Not attempted:** a staged rewrite (prime-sieve for `n−1`, then Sophie-Germain
filter for `n−2`, then factor the rest) would likely be 10–100× faster, since
`n` works only if `n−1` and `(n−2)/2` are both prime. Noted for later.

## [287] Unit-fraction gaps — VERIFIED (Lean reduction + finite check)

**Dead end first.** Direct DFS over gap-≤2 chains with exact rational sums is
exponential; it stalls around max(S) ≈ 40. Recorded as a dead method, kept only
to settle max(S) ≤ 37.

**What worked.** A p-adic valuation argument: if `Σ 1/n = 1` and `T` is the set
of members with maximal `v_p`, then `Σ_{n∈T}(n/p^e)⁻¹ ≡ 0 (mod p)`. For
`p > √b` this forces `p | Σ_{j∈J} L/j`, so for `p > Bound(m)` no multiple of `p`
is in `S` at all. Two *consecutive* excluded integers then contradict "gaps ≤ 2".

**Then the argument was rewritten to be Lean-friendly.** Instead of the lcm of
all of `S`, use the explicit common multiple `D = 2pL` where `L` is the lcm of
the members not divisible by `p`. Every term `D/n` is then visibly a multiple of
`p` except the two special ones, and the whole thing collapses to `p | cL` with
`1 ≤ c ≤ 3 < p`. No valuation machinery at all.

**Formalized** in `ErdosFormal/ErdosFormal/Erdos287.lean` — `elimination`,
`certificate_sub`, `certificate_add`, all with no `sorry` and only
`[propext, Classical.choice, Quot.sound]`.

**Both witness forms turned out to be necessary.** I first formalized only the
`2p−1` case, then checked whether it alone covers every `b`: it does not — there
is a hole at `b = 57`. So `certificate_add` was added to match what the C
covering actually does.

**Off-by-one caught.** `cover287.c` takes `b0` = *first* `b` needing coverage.
Calling it with `37` reports a spurious hole (the first interval starts at 38);
the correct invocation is `./cover287 38 B`. Documented in the source.

Bound reached: max(S) ≤ 10¹⁰ confirmed; a 10¹² covering is running.

## [458] lcm inequality — VERIFIED to 10¹⁴

**Reduction.** `lcm(1..p_{k+1}−1)/lcm(1..p_k)` is *exactly* the product of the
primes `q` having a power `q^e` (`e ≥ 2`) inside the gap `(p_k, p_{k+1})` — there
are no primes in that interval, so every prime power there has `e ≥ 2` and
contributes one extra factor of `q`. The conjecture is literally `∏ q < p_k`.

**Why it is cheap.** Only ~`π(√X)` prime powers exist below `X` (670,121 below
10¹⁴), so walk the prime powers rather than the ~10¹² prime gaps. Grouping two
prime powers into the same gap is decided by *exhibiting* a prime between them,
never by assuming one exists.

Only 5 gaps below 10¹⁴ contain two prime powers — `{8,9}`, `{25,27}`,
`{121,125}`, `{2187,2197}`, `{32761,32768}` — and the tightest case overall is
the very first, `p_k = 7`, ratio 6/7. Runtime 3 minutes.

## [617] Erdős–Gyárfás balanced colourings — DEAD END for the open case

Encoding: `x[e,c]`, exactly-one colour per edge, plus "every `(r+1)`-subset sees
every colour" (equivalently: `K_n` minus each colour class is `K_{r+1}`-free).
Validated against the known `r = 3` theorem: UNSAT.

**Symmetry breaking mattered enormously.** The group is `S_n × S_r` (26!·120 at
r = 5). Sorting the star at vertex 0 — legitimate, since relabelling vertices
1..n−1 is a symmetry — cut r = 3 from **47 s to 0.3 s**, a 150× speedup. A
redundant Turán cardinality constraint (`|colour class| ≥ 55` at r=5) made no
measurable difference.

r = 4, also a known case, did not finish in ~2 hours even so. r = 5 (K₂₆, 5
colours, 1625 vars / 1.15M clauses) is therefore far out of reach for UNSAT, and
r = 5 is the first *open* case. Recorded as a dead end. A SAT answer would still
be trivially checkable — `check_colouring()` re-verifies a claimed colouring from
scratch — so a local-search attempt remains open as future work.

## [699] Erdős–Szekeres gcd of binomials — scanning

`v_p(C(n,i))` is read off as the number of carries adding `i` and `n−i` in base
`p` (Kummer), so no binomial coefficient is ever formed. For each `n` the sets
`A_p = {i : p | C(n,i)}` are bitmasks and the covered-`j` set for a given `i` is
one OR over the relevant primes.

Self-test reproduces two documented facts: `gcd(C(28,5),C(28,14)) = 2³·3³·5`
(with `p = 5` satisfying `p ≥ i` and *no* prime satisfying `p > i`, exactly as the
problem text says), and Sylvester–Schur for all `n < 120`. No counterexample for
`n ≤ 800`; a scan to 3000 is running.

Exact integer test (`n·f(m) < 2·m·f(n)`, no floats). Enumeration restricted to
